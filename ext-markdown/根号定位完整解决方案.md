## ✅ 完整答案

### 核心方法：使用 OpenType MATH 表

Latin Modern Math 等专业数学字体包含 **OpenType MATH 表**，其中存储了：
1. **数学常量** - 根号高度、间距等参数
2. **字形变体** - 不同大小的预制根号
3. **组装部件** - 可拼接的根号组件

## 📐 字体定位能力详解

### 问题：为什么需要"字体定位能力"？

传统的 `drawText()` 只能绘制 Unicode 字符，无法：
- ❌ 访问字形ID（glyph ID）
- ❌ 拉伸单个字形
- ❌ 拼接字形部件
- ❌ 获取字形的精确边界

### 解决：两种方式获得定位能力

#### 方式1：Android 11+ 的 `drawGlyphs()` ⭐

```java
// ✅ 直接绘制字形ID
int[] glyphIds = { 728 };  // 根号的字形ID
float[] positions = { x, y };
canvas.drawGlyphs(glyphIds, 0, positions, 0, 1, mathFont, paint);
```

**优势：**
- 精确控制每个字形
- 可以拉伸、变换
- 性能好

**限制：**
- 只支持 Android 11+ (API 30+)

#### 方式2：预提取 + Path 绘制

```java
// ✅ 预先知道根号的形状参数
// 从 JSON 中读取：
{
  "radical": {
    "variants": [
      { "glyphId": 728, "advance": 800 },  // 小根号
      { "glyphId": 2456, "advance": 1200 }, // 中根号
      { "glyphId": 2457, "advance": 1600 }  // 大根号
    ]
  }
}

// 根据内容高度选择合适的变体
GlyphVariant variant = selectRadicalVariant(contentHeight);

// 使用 Path 按照相同比例绘制
Path path = createRadicalPath(variant.advance);
canvas.drawPath(path, paint);
```

**优势：**
- 支持所有 Android 版本
- 不依赖系统API

**限制：**
- 效果略逊于真实字形
- 需要预处理

## 🔧 实现细节

### 1. 提取字体数据（Python）

```python
from fontTools.ttLib import TTFont

font = TTFont('latinmodern-math.otf')
math_table = font['MATH'].table

# 获取根号字形ID
cmap = font.getBestCmap()
radical_glyph_id = cmap[0x221A]  # √

# 提取垂直变体
vert_construction = math_table.MathGlyphInfo. \
    MathGlyphConstruction.VertGlyphConstruction[radical_glyph_id]

for variant in vert_construction.MathGlyphVariantRecord:
    print(f"变体: ID={variant.VariantGlyph}, 高度={variant.AdvanceMeasurement}")

# 提取部件
if vert_construction.GlyphAssembly:
    for part in vert_construction.GlyphAssembly.PartRecords:
        print(f"部件: ID={part.glyph}, 高度={part.FullAdvance}, "
              f"可重复={bool(part.PartFlags & 0x0001)}")
```

### 2. 加载数据（Android）

```java
public class LatinModernMathData {
    public int unitsPerEm;  // 字体单位，如 1000
    public RadicalData radical;
    
    // 转换字体单位到像素
    public float toPixels(int fontUnits, float fontSize) {
        return fontUnits * fontSize / unitsPerEm;
    }
    
    // 选择合适的变体
    public GlyphVariant selectRadicalVariant(float targetHeight) {
        float targetHeightFU = targetHeight * unitsPerEm / fontSize;
        
        for (GlyphVariant v : radical.variants) {
            if (v.advance >= targetHeightFU) {
                return v;
            }
        }
        
        return radical.variants.get(radical.variants.size() - 1);
    }
}
```

### 3. 精确定位（关键！）

```java
public void renderRadical(Canvas canvas, float x, float y,
                         float contentHeight, float contentDepth,
                         float fontSize) {
    // 1. 从 MATH 表获取参数
    float gap = mathData.toPixels(mathData.constants.radicalVerticalGap, fontSize);
    float ruleThickness = mathData.toPixels(mathData.constants.radicalRuleThickness, fontSize);
    float extraAscender = mathData.toPixels(mathData.constants.radicalExtraAscender, fontSize);
    
    // 2. 计算所需高度
    float targetHeight = contentHeight + contentDepth + gap + ruleThickness + extraAscender;
    
    // 3. 选择合适的变体
    GlyphVariant variant = mathData.selectRadicalVariant(targetHeight);
    
    // 4. 定位绘制
    float radicalTop = y - contentHeight - gap - ruleThickness - extraAscender;
    float radicalBottom = y + contentDepth;
    
    // 绘制根号
    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.R) {
        // 方式1：直接绘制字形
        canvas.drawGlyphs(new int[]{variant.glyphId}, ...);
    } else {
        // 方式2：Path 绘制
        Path path = createRadicalPath(radicalTop, radicalBottom);
        canvas.drawPath(path, paint);
    }
}
```

## 📊 定位参数说明

### 关键 MATH 常量

| 参数 | 说明 | 典型值 | 用途 |
|------|------|--------|------|
| `RadicalVerticalGap` | 根号到内容的间距 | 50 | 避免根号贴内容 |
| `RadicalRuleThickness` | 上横线粗细 | 40 | 绘制上横线 |
| `RadicalExtraAscender` | 根号额外高度 | 80 | 根号超出内容 |
| `RadicalKernBeforeDegree` | 根指数左间距 | 100 | n次根的n的位置 |
| `RadicalKernAfterDegree` | 根指数右间距 | 50 | n和根号的间距 |
| `RadicalDegreeBottomRaisePercent` | 根指数抬升比例 | 65% | n的垂直位置 |

### 坐标系统

```
y - contentHeight - extraAscender  ←─ 根号顶部
    │
    ├─ extraAscender (根号额外高度)
    │
y - contentHeight - gap - ruleThickness  ←─ 上横线位置
    │
    ├─ ruleThickness (线粗细)
    │
y - contentHeight - gap  ←─ 上横线底部
    │
    ├─ gap (间距)
    │
y - contentHeight  ←─ 内容顶部
    │
    ├─ contentHeight (内容高度)
    │
y  ←─ 基线 (baseline)
    │
    ├─ contentDepth (内容深度)
    │
y + contentDepth  ←─ 内容底部 / 根号底部
```
